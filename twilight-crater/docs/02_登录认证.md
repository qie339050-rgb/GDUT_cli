# 02 — 登录认证

## 登录方式对比

| 方式 | 推荐度 | 说明 |
|------|--------|------|
| CAS 统一认证 | ⭐⭐⭐ 推荐 | 使用学校统一身份认证，最通用 |
| 本地帐号登录 | ⭐ 备用 | 需要验证码，较不方便 |

---

## 方式一：CAS 统一认证（推荐）

### 流程概览

```
1. GET 登录页 → 获取 execution、pwdEncryptSalt
2. AES-CBC 加密密码
3. POST 提交表单 → 302 重定向
4. 跟随重定向 → 到达教务系统首页
5. 保存 JSESSIONID Cookie
```

### Step 1: 获取登录页

```http
GET https://authserver.gdut.edu.cn/authserver/login?service=https%3A%2F%2Fjxfw.gdut.edu.cn%2Fnew%2FssoLogin
```

从 HTML 中提取以下隐藏字段：

| 字段 | 来源 | 说明 |
|------|------|------|
| `execution` | `<input name="execution" value="...">` | 流程控制 Token，每次请求不同 |
| `pwdEncryptSalt` | `<input id="pwdEncryptSalt" value="...">` | AES 加密的 Key |

**示例 execution 值**：`34a48fa3-6a2b-4d89-8ff1-1eef36708673_ZXlK...`（很长的字符串）

### Step 2: AES-CBC 加密密码

从 `encrypt.js` 提取的完整加密逻辑：

```
算法: AES-CBC
Padding: PKCS7
Key: pwdEncryptSalt（UTF-8 编码，16字节）
IV: 随机生成的 16 位字符串（UTF-8 编码）
明文: 随机64位字符串 + 原始密码
输出: Base64 编码
```

**可用字符集**（用于生成随机字符串）：
```
ABCDEFGHJKMNPQRSTWXYZabcdefhijkmnprstwxyz2345678
```

> 注意：故意排除了易混淆字符 I, L, O, U, V, g, l, o, q, u, v, 0, 1, 9

**Python 实现参考**：

```python
from Crypto.Cipher import AES
from Crypto.Util.Padding import pad
import base64, random, string

CHARSET = 'ABCDEFGHJKMNPQRSTWXYZabcdefhijkmnprstwxyz2345678'

def random_string(length):
    return ''.join(random.choice(CHARSET) for _ in range(length))

def encrypt_password(password, salt):
    random_iv = random_string(16)
    random_prefix = random_string(64)
    plaintext = random_prefix + password

    key = salt.encode('utf-8')
    iv = random_iv.encode('utf-8')

    cipher = AES.new(key, AES.MODE_CBC, iv)
    encrypted = cipher.encrypt(pad(plaintext.encode('utf-8'), AES.block_size))
    return base64.b64encode(encrypted).decode('utf-8')
```

### Step 3: POST 提交登录

```http
POST https://authserver.gdut.edu.cn/authserver/login?service=https%3A%2F%2Fjxfw.gdut.edu.cn%2Fnew%2FssoLogin
Content-Type: application/x-www-form-urlencoded

username={学号}
&password={加密后的密码}
&captcha=
&_eventId=submit
&cllt=userNameLogin
&dllt=generalLogin
&lt=
&execution={execution值}
```

### Step 4: 跟随重定向

登录成功后：
```
302 → https://jxfw.gdut.edu.cn/new/ssoLogin?ticket=ST-XXXXXX
302 → https://jxfw.gdut.edu.cn/login!welcome.action
200 ← 教务系统首页 HTML
```

此时 `requests.Session` 中的 cookies 已包含有效的 `JSESSIONID`。

### 登录失败判断

| 情况 | 判断方法 |
|------|---------|
| 密码错误 | 响应 HTML 中包含 `"用户名或密码不正确"` |
| 需要验证码 | 响应 HTML 中验证码区域可见 |
| 账号锁定 | 响应 HTML 中包含 `"锁定"` |
| 登录成功 | 最终 URL 包含 `welcome.action` |

### 验证码相关接口

```http
GET https://authserver.gdut.edu.cn/authserver/checkNeedCaptcha.htl?username={学号}&_={timestamp}
```

返回 JSON：`{"isNeed": true/false}`

若需要验证码：
```http
GET https://authserver.gdut.edu.cn/authserver/captcha.html?ts={timestamp}
```

返回验证码图片。

## 方式二：本地帐号登录（备用）

### 验证码获取

```http
GET https://jxfw.gdut.edu.cn/yzm?d={timestamp}
```

返回验证码图片。

### 加密方式

```
算法: AES-ECB
Padding: PKCS7 (默认)
Key: 验证码重复4次（如验证码 "1234" → Key 为 "1234123412341234"）
明文: 原始密码
输出: 加密后的字符串
```

### 登录提交

```http
POST https://jxfw.gdut.edu.cn/new/login
Content-Type: application/x-www-form-urlencoded

account={学号}
&pwd={AES-ECB加密后的密码}
&verifycode={验证码}
```

## Session 管理

### 保存 Session

登录成功后，将 cookies 保存到本地文件（如 `~/.jw/session.json`）：

```json
{
  "cookies": {
    "JSESSIONID": "xxxxxxxxxxxxxxxx"
  },
  "saved_at": "2026-03-02T20:00:00",
  "base_url": "https://jxfw.gdut.edu.cn"
}
```

### 复用 Session

后续请求时加载 cookies，设置到 `requests.Session` 中。

### 检测 Session 过期

任何接口请求后检查：
- 响应状态码是否为 302
- 最终 URL 是否包含 `login` 或 `authserver`
- 响应内容是否包含 `"请求超时"` 或登录表单

若过期，提示用户重新执行 `jw login`。
